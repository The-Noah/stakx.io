Understanding Stakx

To_config.yml or not to_config.yml

The _config.yml is completely optional and a website can be built with the --no-config option. A configuration file is extremely useful and is recommended for a production-ready website but if it's quick and dirty, then you don't need one.

Stakx Data Types

Collections

A Collection is a group of ContentItems. As an example, when building a blog, all of the posts together would be in a collection with an arbitrary name such as "posts."

ContentItems

A Contentitem is any element in your website that belongs in a collection. Continuing with the blog example, each post would become a ContentItem. Each ContentItem contains FrontMatter which allows you to specify information regarding the ContentItem such as author, categories, tags, and date posted.

DataSet

A DataSet is a group of DataItems

FrontMatter

FrontMatter is a section at the top of each ContentItem or PageView denoted by '...' with information such as variables or special values to trigger stakx behavior.

PageViews

A PageView is a Twig template with FrontMatter, Specific FrontMatter values have special meaning such as:
-pagination
-categories
-permalink
-title
